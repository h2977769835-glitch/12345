# 🚀 RQ-VAE (Residual Quantized VAE) 深度训练框架

本项目提供了一个基于 PyTorch 生态构建的高效 **RQ-VAE（残差量化变分自编码器）** 完整训练流水线。

在当前的生成式检索（Generative Retrieval）、推荐系统以及高维特征压缩任务中，如何将连续的、高维的稠密向量（Dense Embeddings）转化为极度紧凑且具有高度区分度的离散语义索引（Discrete Codes）是一项核心挑战。本框架正是为解决这一痛点而生，它通过引入多层级的残差量化机制，结合 K-Means 质心初始化、Sinkhorn 聚类对齐以及灵活定制的 MLP 网络架构，实现了高效的端到端表征压缩与离散化。

---

## ✨ 核心特性与架构亮点

- **层级残差量化 (Hierarchical Residual Quantization):** 打破传统单次 VQ-VAE 的量化瓶颈。通过引入多级级联码本（Codebooks），每一层专门负责拟合上一层的量化残差，从而在极小的码本规模下实现对原始向量的高精度逼近，显著降低量化误差。
- **高度模块化的网络拓扑:**
  - **Encoder / Decoder:** 基于多层感知机（MLP）构建，用户可通过简单的传参自由定制隐层的深度与节点维度。
  - **Codebook 空间:** 支持对每一层量化器的码本容量（`num_emb_list`）和词向量维度（`e_dim`）进行异构配置，满足不同颗粒度的压缩需求。
- **工业级初始化与死码消除:**
  - **K-Means 预热:** 抛弃随机初始化，采用 K-Means 对数据分布进行预先聚类以初始化码本质心，大幅缩短模型早期的收敛时间。
  - **Sinkhorn 最优传输:** 可选集成 Sinkhorn 算法进行特征对齐，强制平衡各个 Code 的使用频率，从根本上缓解传统 VQ 模型中常见的“Index Collapse（死码/特征坍塌）”难题。
- **多维度指标监控体系:** 除了传统的均方误差（MSE）和重建损失（Reconstruction Loss），框架创新性地引入了 **冲突率 (Collision Rate)** 监控。该指标直观反映了量化后的离散码序列能否唯一且无歧义地标识不同的原始样本。
- **全自动训练管线:** 原生集成 TensorBoard 训练可视化、学习率预热机制 (Warmup)、防止梯度爆炸的梯度裁剪 (Gradient Clipping)，以及基于自定义指标（如最低 Loss 或最低冲突率）的最佳模型自动容灾保存机制。

## 💾 数据流与格式准备

本框架的数据输入管道 (`datasets.py` 内的 `CustomNpzFile` 类) 针对 NumPy 的 `.npz` 压缩格式进行了高度优化。

- **挂载目录:** 启动训练时，需通过 `--data_path` 参数指定存放训练数据的父级目录。
- **规范字段:** 目录下所有的 `.npz` 文件均会被自动扫描并拼接，每个文件内部必须严格包含以下键值对（Keys）：
  - `embs`: 模型输入的核心稠密向量矩阵（推荐使用 `Float32` 数据类型）。
  - `ids`: 样本的唯一标识符（虽不直接参与反向传播和 Loss 计算，但对后期的索引哈希表构建与碰撞率分析至关重要）。

## 🚀 极速启动指南

### 1. 配置并触发训练

您可以通过修改 `run.sh` 脚本，或者直接在终端中执行带有详细参数的命令来拉起训练进程：

```bash
# 启动训练的标准化命令示例
python -u rqvae_train.py \
  --data_path "/path/to/your/npz/data_directory" \
  --ckpt_dir "./output_checkpoints" \
  --batch_size 8192 \
  --lr 3e-4 \
  --epochs 1000 \
  --kmeans_init True \
  --loss_type "mse"
```

### 2. 核心超参数全解

在 `rqvae_train.py` 的解析器中，提供了丰富的旋钮供开发者调优：

**📐 架构与容量维度:**
- `--layers`: 决定 Encoder 和 Decoder 复杂度的隐层神经元列表。默认配置为 `[4096, 2048, 1024, 512, 256, 128, 64]`，呈漏斗状压缩。
- `--num_emb_list`: 决定各层残差量化码本容量的列表。例如 `[2048, 2048, 1024]` 意味着启用 3 级级联，前两级有 2048 个候选聚类中心，最后一级 1024 个。
- `--e_dim`: 单个离散特征（Code）背后的隐向量维度。默认值为 `32`。
- `--dropout_prob`: 正则化丢弃率，用于防止网络过拟合。

**🔥 损失与目标函数:**
- `--loss_type`: 衡量重构质量的度量方式，支持 `mse` (L2 Loss) 或 `l1` (MAE Loss)。
- `--quant_loss_weight`: 总 Loss 中量化误差项所占的比重。默认为 `1.0`。
- `--beta`: 承诺损失 (Commitment Loss) 的缩放系数，用于约束 Encoder 输出不至于偏离码本空间太远。默认为 `0.25`。
- `--kmeans_init`: 布尔值，强烈建议开启 (`True`)，利用传统聚类算法为深度网络提供极佳的起始权重。
- `--sk_epsilons`: 控制 Sinkhorn 算法熵正则化强度的 epsilon 列表，需与码本层数对齐。

**⚙️ 优化器与调度策略:**
- `--learner`: 求解器选项，支持深度学习主流算法（如 `AdamW`, `Adam`, `SGD`）。
- `--lr`: 基础学习率（Learning Rate）。
- `--lr_scheduler_type`: 学习率退火策略，可选择 `constant` (恒定) 或 `linear` (线性衰减)。
- `--warmup_epochs`: 训练初期的预热轮数，有助于模型在寻找极小值时更加稳定。

## 📊 监控指标与资产输出

模型在迭代周期内，会自动在您指定的目录生成以下资产：

1. **终端实时回显:** 滚动播报当前进度（Epoch）、训练总误差（Training Loss）、向量重建误差（Reconstruction Loss）以及验证集独有的冲突率（Collision Rate）。
2. **TensorBoard 遥测图表:** 所有标量数据实时同步至 `./logs` 目录，供可视化面板读取：
   - `Step/loss_total`: 包含重构与量化惩罚的综合误差趋势。
   - `Step/loss_recon`: 纯粹衡量信号还原度的误差。
   - `Epoch/collision_rate`: 验证阶段计算的索引碰撞概率。
3. **模型快照 (Checkpoints):** 序列化后的权重文件保存在 `--ckpt_dir` 设定路径：
   - `best_loss_model.pth`: 验证集上综合误差打破历史最低记录的权重。
   - `best_collision_model.pth`: 语义索引区分度最高（冲突率最低）的权重。
   - `epoch_x_collision_y.pth`: 按照特定频率保留的中间态模型，方便进行断点续训或回溯。

## 🧩 RQ-VAE 数学与算法推演简述

理解 RQ-VAE，核心在于掌握 **残差向量量化器 (Residual Vector Quantizer, RVQ)** 的级联逻辑。

1. **特征编码 (Encoder):** 首先将高维连续空间中的输入样本 $x$ 映射至低维潜在特征空间，得到潜向量 $z$。
2. **残差量化瀑布流 (RVQ):**
   - 步骤 1：第 1 层量化器在自身的码本中寻找与 $z$ 最近邻的向量 $z_1$，并计算未被拟合的残差 $r_1 = z - z_1$。
   - 步骤 2：第 2 层量化器接收上一层的残差 $r_1$ 作为输入，找到最近邻的 $z_2$，再次计算新的残差 $r_2 = r_1 - z_2$。
   - 步骤 N：依此类推，进行多轮逼近。
   - 最终聚合：该样本在潜在空间的最终量化表达为所有层级的向量之和，即 $\hat{z} = \sum_{i=1}^{L} z_i$。
3. **特征解码 (Decoder):** 接收组合后的近似潜向量 $\hat{z}$，通过上采样网络尽力将其重构回原始维度的 $\hat{x}$。

**关于冲突率 (Collision Rate) 的技术补充:**
代码在 `trainer.py` 的 `_valid_epoch` 验证流中进行了专项统计。当两条截然不同的输入数据在经过上述 RVQ 瀑布流后，如果恰好命中了完全一样的多级索引组合（如都生成了 `[4, 102, 56]` 这样的离散序列），这就产生了一次“冲突”。冲突率越低，说明当前的量化空间越丰富，生成的离散索引对特征的表征能力越精准。
